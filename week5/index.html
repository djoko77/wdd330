Chapter 10: Testing and Debugging

  • Errors are caused when something goes wrong in a program: system, programmer, user
  • Exception - error that produces a return value that can then be used by the program to deal with the error
  • Stack trace - sequence of functions or method calls that lead to the point where the error occurred
  • Warning - occurs if thereâ€™s an error in the code that isn't enough to cause the program to crash
  • Strict mode - produces more exceptions and warnings and prohibits the use of some deprecated features ( 'use strict'; )
  • Linting tools - test the quality of JavaScript code
  • Debugging - the process of finding out where bugs occur in the code
  • alert() method to show a dialog at certain points in the code
  • console.log() method is used to log the value of variables at different stages of the program
  • console.trace() method logs an interactive stack trace in the console
  • The code inside the catch block will only run if an exception is thrown inside the try block
  • Finally block will always be executed after the try or catch block, regardless of whether an exception occurred or not
  • Test-driven development (TDD) is the process of writing tests before any actual code
